## Clarifications

### Session 2026-04-02

- Q: 앱 윈도우 모델 (팝오버 vs 메인 윈도우)? → A: 메인 윈도우 전용. 메뉴바 아이콘은 앱 열기만 담당하고, 모든 기능은 메인 윈도우에서 제공한다.
- Q: MVP 뷰 모드 범위 (Calendar/List/Timesheet)? → A: 캘린더 뷰만 지원. 리스트 뷰와 타임시트는 MVP 범위 외.
- Q: 캘린더 시간 블록 드래그 인터랙션? → A: 드래그 없음. 시간 블록 클릭 시 인라인/팝업으로 편집.
- Q: 시간 항목 삭제 방식? → A: 삭제 가능 + ⌘Z Undo 지원.
- Q: 메인 윈도우 사이드바 구성? → A: 최소 사이드바. 캘린더 뷰 + 프로젝트 목록/관리만 포함.
- Q: Timer와 TimeEntry의 관계? → A: Timer는 별도 엔티티. 실행 중 상태를 별도 저장하고 정지 시 TimeEntry로 변환한다.
- Q: 캘린더 표시 시간 범위? → A: 24시간 전체(0:00~24:00) 항상 표시.
- Q: 주간 뷰 시작 요일? → A: 월요일 시작.
- Q: Undo 범위? → A: 삭제만 Undo 지원. 편집/생성은 Undo 대상이 아니다.
- Q: 키보드 단축키 범위? → A: 타이머 시작/정지만 글로벌 단축키(앱 비활성 시에도 동작). 나머지(⌘N 수동입력, ⌘Z 삭제취소 등)는 앱 내 단축키.

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 타이머 시작과 정지 (Priority: P1)

사용자가 업무를 시작할 때 타이머를 시작하고, 업무가 끝나면 정지하여 시간을 기록한다. 메인 윈도우 상단에 타이머 컨트롤이 위치하며, 원클릭으로 타이머를 시작/정지할 수 있다. 타이머가 실행 중일 때 메뉴바 아이콘과 메인 윈도우에 경과 시간이 실시간으로 표시된다.

**Why this priority**: 시간 추적 앱의 가장 핵심 기능. 이 기능 없이는 앱의 존재 이유가 없다.

**Independent Test**: 앱을 실행하고 타이머를 시작/정지하여 시간이 기록되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 타이머가 정지 상태일 때, **When** 사용자가 시작 버튼을 클릭하면, **Then** 타이머가 시작되고 메뉴바 아이콘과 메인 윈도우에 경과 시간(00:00:00)이 표시된다
2. **Given** 타이머가 실행 중일 때, **When** 사용자가 정지 버튼을 클릭하면, **Then** 타이머가 정지되고 경과 시간이 시간 항목(time entry)으로 저장된다
3. **Given** 타이머가 실행 중일 때, **When** 앱을 종료하고 다시 실행하면, **Then** 타이머가 이전 상태를 유지하며 계속 실행된다
4. **Given** 메뉴바 아이콘을 클릭하면, **When** 앱이 활성화되고, **Then** 메인 윈도우가 표시된다

---

### User Story 2 - 업무명 등록 및 수정 (Priority: P2)

사용자가 시간 항목에 업무명(description)을 지정하여 어떤 작업에 시간을 사용했는지 기록한다. 타이머 시작 전/후에 업무명을 입력하거나 수정할 수 있다. 프로젝트를 지정하여 업무를 분류할 수 있다.

**Why this priority**: 단순히 시간만 기록하면 의미가 없다. 어떤 업무인지 식별할 수 있어야 시간 추적의 가치가 생긴다.

**Independent Test**: 타이머를 시작하면서 업무명을 입력하고, 기록된 항목에서 업무명이 올바르게 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 메인 윈도우에서 타이머 영역이 보일 때, **When** 업무명 입력 필드에 텍스트를 입력하고 타이머를 시작하면, **Then** 해당 업무명과 함께 타이머가 시작된다
2. **Given** 타이머가 실행 중일 때, **When** 업무명을 수정하면, **Then** 현재 실행 중인 시간 항목의 업무명이 즉시 업데이트된다
3. **Given** 저장된 시간 항목이 캘린더에 있을 때, **When** 해당 시간 블록을 클릭하면, **Then** 편집 패널이 나타나고 업무명을 수정할 수 있다
4. **Given** 편집 패널이 열려 있을 때, **When** 프로젝트 선택에서 프로젝트를 선택하면, **Then** 시간 항목이 해당 프로젝트에 분류된다

---

### User Story 3 - 캘린더 뷰에서 시간 항목 확인 (Priority: P3)

사용자가 캘린더 형태의 뷰에서 기록된 시간 항목을 시각적으로 확인한다. 일간/주간 뷰에서 시간 블록이 타임라인에 배치되어 하루의 시간 사용 패턴을 직관적으로 파악할 수 있다. 좌측 최소 사이드바에서 프로젝트 목록을 관리할 수 있다.

**Why this priority**: 캘린더 뷰는 기록된 데이터를 의미있게 시각화하는 핵심 UI이지만, 타이머와 업무명 기능이 먼저 있어야 표시할 데이터가 존재한다.

**Independent Test**: 여러 시간 항목을 기록한 후 캘린더 뷰에서 시간 블록이 올바른 위치에 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 오늘 기록된 시간 항목이 있을 때, **When** 캘린더 일간 뷰를 열면, **Then** 각 시간 항목이 해당 시간대에 블록으로 표시된다
2. **Given** 캘린더 뷰가 열려 있을 때, **When** 주간 뷰로 전환하면, **Then** 월요일부터 일요일까지 7일간의 시간 블록이 컬럼별로 표시된다
3. **Given** 캘린더 뷰에서 시간 블록을 클릭하면, **When** 편집 패널이 나타나고, **Then** 업무명, 프로젝트, 시간을 편집할 수 있다
4. **Given** 캘린더 뷰가 열려 있을 때, **When** 날짜를 이전/다음으로 이동하면, **Then** 해당 날짜의 시간 항목이 표시된다
5. **Given** 타이머가 실행 중이고 캘린더 뷰가 열려 있을 때, **When** 시간이 경과하면, **Then** 현재 실행 중인 시간 항목의 블록이 실시간으로 확장되어 경과 시간을 반영한다

---

### User Story 4 - 수동 시간 입력 (Priority: P4)

사용자가 타이머를 사용하지 않고 과거의 작업 시간을 수동으로 입력한다. 캘린더 뷰에서 빈 시간대를 클릭하여 수동 입력 폼을 통해 시작/종료 시간과 업무명을 직접 지정할 수 있다.

**Why this priority**: 타이머를 깜빡했거나 과거 작업을 소급 기록해야 하는 경우에 필요한 보조 기능이다.

**Independent Test**: 수동 입력 폼에서 시작/종료 시간과 업무명을 입력하고, 캘린더 뷰에 해당 항목이 올바르게 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 캘린더 뷰에서 빈 시간대를 클릭할 때, **When** 수동 입력 폼이 나타나면, **Then** 시작 시간이 클릭한 시간대로 자동 설정된다
2. **Given** 수동 입력 폼이 열려 있을 때, **When** 시작/종료 시간, 업무명, 프로젝트를 입력하고 저장하면, **Then** 새 시간 항목이 생성되고 캘린더에 표시된다
3. **Given** 수동 입력 시 시작 시간이 종료 시간보다 늦을 때, **When** 저장을 시도하면, **Then** 유효성 오류 메시지가 표시되고 저장이 거부된다
4. **Given** 수동 입력 시 기존 시간 항목과 겹치는 시간을 입력할 때, **When** 저장을 시도하면, **Then** 겹침 경고가 표시되고 사용자가 계속 진행할지 선택할 수 있다

---

### Edge Cases

- 타이머가 실행 중인 상태에서 날짜가 바뀌면(자정 경과) 시간 항목이 하나의 연속 항목으로 유지된다
- 업무명 없이 타이머를 시작하고 정지한 경우 "제목 없음" 항목으로 저장된다
- 매우 짧은 시간(1분 미만) 항목도 유효한 기록으로 저장된다
- 시스템 절전(sleep) 모드에서 깨어났을 때 타이머가 절전 시간을 포함하여 정확한 경과 시간을 표시한다
- 프로젝트가 삭제된 경우 해당 프로젝트에 속한 시간 항목은 "프로젝트 없음"으로 변경된다
- 시간 항목 삭제 후 ⌘Z로 되돌리기가 가능하다

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 메뉴바에 타이머 아이콘을 표시하고 클릭 시 메인 윈도우를 활성화해야 한다
- **FR-002**: 메인 윈도우 상단에 타이머 시작/정지 컨트롤을 제공해야 하며, 실행 중 경과 시간을 메뉴바 아이콘과 메인 윈도우에 실시간 표시해야 한다
- **FR-003**: 타이머 정지 시 시간 항목(시작 시간, 종료 시간, 경과 시간)을 자동으로 로컬에 영속 저장해야 한다
- **FR-004**: 사용자는 시간 항목에 업무명(description)을 지정, 수정할 수 있어야 한다
- **FR-005**: 사용자는 시간 항목에 프로젝트를 지정할 수 있어야 한다
- **FR-006**: 사용자는 프로젝트를 생성, 수정, 삭제할 수 있어야 한다. 프로젝트는 이름과 색상을 가진다
- **FR-007**: 일간/주간 캘린더 뷰를 제공하여 시간 항목을 시간 블록으로 시각화해야 한다. 세로축은 24시간 전체(0:00~24:00)를 표시한다. 타이머가 실행 중일 때 해당 시간 블록은 경과 시간에 따라 실시간으로 확장되어야 한다
- **FR-008**: 사용자는 캘린더 뷰에서 날짜를 이동(이전/다음)할 수 있어야 한다
- **FR-009**: 사용자는 수동으로 시작/종료 시간을 지정하여 시간 항목을 생성할 수 있어야 한다
- **FR-010**: 수동 입력 시 시간 유효성(시작 < 종료)을 검증해야 한다
- **FR-011**: 앱 종료 후 재실행 시 실행 중이던 타이머 상태를 복원해야 한다
- **FR-012**: 겹치는 시간 항목 입력 시 사용자에게 경고를 표시해야 한다
- **FR-013**: 사용자는 시간 항목을 삭제할 수 있어야 하며, ⌘Z로 삭제를 되돌릴 수 있어야 한다. Undo는 삭제 작업에만 적용되며, 편집/생성은 Undo 대상이 아니다
- **FR-014**: 메인 윈도우 좌측에 최소 사이드바를 제공하여 프로젝트 목록 조회 및 관리 기능에 접근할 수 있어야 한다
- **FR-015**: 캘린더 뷰에서 시간 블록을 클릭하면 편집 패널이 나타나 업무명, 프로젝트, 시간을 수정할 수 있어야 한다 (드래그 이동/크기 조절은 지원하지 않는다)
- **FR-016**: 타이머 시작/정지를 위한 글로벌 키보드 단축키를 제공해야 한다 (앱이 비활성 상태에서도 동작)
- **FR-017**: 앱 내 키보드 단축키를 제공해야 한다 (⌘N 수동 시간 입력, ⌘Z 삭제 취소 등)

### Key Entities

- **TimeEntry**: 하나의 시간 기록. 시작 시간, 종료 시간(실행 중이면 null), 업무명, 프로젝트 참조를 가진다
- **Project**: 업무 분류 단위. 이름과 색상을 가지며, 여러 TimeEntry에 연결될 수 있다
- **Timer**: 현재 실행 중인 타이머 상태를 관리하는 별도 엔티티. 시작 시간, 업무명, 프로젝트 참조를 가진다. 타이머 정지 시 Timer 데이터를 기반으로 TimeEntry를 생성하고 Timer를 초기화한다. 동시에 하나만 존재할 수 있다

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 사용자가 메인 윈도우에서 타이머를 시작하기까지 1클릭으로 가능하다
- **SC-002**: 타이머 시작 후 메뉴바와 메인 윈도우에 경과 시간이 1초 이내에 표시된다
- **SC-003**: 캘린더 뷰에서 하루 50개 이상의 시간 항목을 렌더링해도 스크롤이 부드럽다(60fps)
- **SC-004**: 앱 재실행 시 이전 타이머 상태가 복원되기까지 2초 이내이다
- **SC-005**: 수동 시간 입력을 30초 이내에 완료할 수 있다
- **SC-006**: 6개월 이상의 데이터(일 평균 10개 항목)를 저장해도 앱 시작 시간이 3초를 넘지 않는다
- **SC-007**: 삭제된 시간 항목을 ⌘Z로 즉시 복원할 수 있다

## Assumptions

- 단일 사용자 앱이다 (다중 사용자/계정 전환 없음)
- macOS 전용이며 iOS/iPadOS 지원은 범위 외이다
- 네트워크 연결이 불필요하다 (로컬 전용, 클라우드 동기화 없음)
- 인증/로그인이 불필요하다 (로컬 앱)
- 캘린더 뷰는 일간/주간만 제공하며, 월간 뷰는 MVP 범위 외이다
- 리스트 뷰, 타임시트 뷰는 MVP 범위 외이다
- 리포트/통계 기능은 MVP 범위 외이다
- 데이터 내보내기(CSV/JSON)는 MVP 범위 외이나 향후 추가 가능하다
- 타이머는 동시에 하나만 실행 가능하다 (다중 타이머 불가)
- 자정 경과 시 시간 항목을 자동 분리하지 않고 하나의 연속 항목으로 유지한다
- 시간 블록은 드래그 이동/크기 조절을 지원하지 않으며, 클릭하여 편집한다
- 메인 윈도우 전용 앱이며, 메뉴바 아이콘은 앱 활성화 용도로만 사용한다
